Week 2 Notes

Chapter 2 Readings:

  • Var is typically used in older programming as its scope is global
  • Let was introduced in ES6, but couldn’t modify behavior of Var due to backwards compatibility
  • Block scope = const and let, value only exists in block they are declared
  • Global scope = var, accessible everywhere
  • Blocks {} can be used to make local scope, local scope means the variable is only accessible in that block
  • Backslash guide:
  • Single quote marks: \'
  • Double quote marks: \"
  • End of line: \n
  • Carriage return: \r
  • Tab: \t
  • Convert number to a string: string(9) or 9..toString();
  • String to int: parseInt(‘108’) you can also specify the base 2, 36, 10
  • Null: no value
  • Undefined: NaN
  • Lazy Evaluation:
  • let a = 0
  • false && (a=1) << false
  • false || (a = 1) << 1 //a =1 now
  • === : tests for equality if they are the same data type
  • == : tests for equality without taking for account data type
  • 0 is falsy value

Chapter 3 Readings:

  • pop() – removes last item in array
  • shift() – removes first item in array
  • push(value) – puts value at end of array
  • unshift(value) – appends item to beginning of the array
  • concat(array, array) – merges the two arrays together
  • join() – stringify array
  • splice(x, y) – cuts array between the two indexes
  • splice(x, y, value) – cuts array similar to slice() and puts value in place
  • reverse() – reverses order
  • sort() – sorts alphabetically
  • ... - spread syntax
  • const flat = […summer, …winter] << [combined arrays]
  • set() = useful for checking data, as any duplicate data is ignored
  • weakset() – garbage collects any removed objects
  • map() – similar to dictionary with key, pair values
  • temery operator - ?
